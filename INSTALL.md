# Installing nimrod (development)

This is the canonical, tested install path. The `README.md` shows only
`pip install slg-nimrod`, which installs the published library but **not** a
working dev/recipe environment ‚Äî use this file for that.

> `README.md` is auto-generated by nbdev from `nbs/index.ipynb`; don't edit it
> by hand. This file (`INSTALL.md`) is the source of truth for install steps.

## TL;DR (macOS, Apple Silicon)

```bash
# 1. create + activate the env (Python 3.11, pinned torch stack)
micromamba env create -f .devcontainer/environment-macos.yml   # creates env "nimrod"
micromamba activate nimrod

# 2. install the library + its Python deps (editable)
pip install -e .

# 3. verify
python -c "import nimrod, torch, lightning, torchvision; print('torch', torch.__version__)"
```

That's enough to run the **image recipes** (e.g. MNIST). Audio/speech and
text/LM recipes need the extra steps below.

> Substitute `conda`/`mamba` for `micromamba` if you prefer ‚Äî the commands are
> identical. The env file names the environment `nimrod`.

## Why torch is pinned to 2.3.0

The whole torch family is pinned to the **2.3.0 release matrix**:

| package      | version |
|--------------|---------|
| torch        | 2.3.0   |
| torchvision  | 0.18.0  |
| torchaudio   | 2.3.0   |
| torchtext    | 0.18.0  |
| transformers | `<5`    |
| numpy        | `<2`    |

Three hard constraints force this and they are not negotiable while the code
uses `torchtext`:

1. **torchtext is archived.** Its final release is `0.18.0`, which supports
   **only torch 2.3.x**. `nimrod/text/datasets.py` imports it, so the text
   recipes require it.
2. **torch 2.3 predates NumPy 2.** Importing torch 2.3 against `numpy>=2` raises
   the "compiled using NumPy 1.x" error, so numpy is pinned `<2`.
3. **transformers 5.x requires torch ‚â•2.4.** Newer transformers silently
   disables its PyTorch backend on torch 2.3 ("Disabling PyTorch because
   PyTorch >= 2.4 is required"), which breaks every transformers-based model
   (BLIP, CLIP, ‚Ä¶). So transformers is pinned `<5` (resolves to the 4.57.x
   line, which works with torch 2.3).

Keep the four torch packages in lock-step ‚Äî bumping one without the others
breaks the ABI. If you don't need the text recipes, you *can* drop `torchtext`
and move to a newer torch, but that's a code change (refactor
`nimrod/text/datasets.py`), not just an install tweak.

## Tiers

`pip install -e .` (step 2) already installs the deps for these tiers ‚Äî the
extra work below is only the **native** (non-pip) pieces.

### Core ‚Äî image recipes
No extra native deps. After the TL;DR you can run, e.g.:

```bash
python train.py experiment=mnist_mlp trainer.max_epochs=5
```

### Audio / speech recipes
The Python deps (`torchaudio`, `lhotse`, `encodec`, `librosa`) come from
`pip install -e .`. The native codecs (`ffmpeg`, `libsndfile`) are already in
`environment-macos.yml`, so no extra step is needed for most audio recipes.

`phonemizer` (TTS / text normalization) additionally needs the **espeak**
system library ‚Äî see the next tier.

### Text / LM recipes (kenlm + espeak)
Only needed for n-gram LMs (`nimrod/models/ngram.py`) and phoneme-based recipes.
With the `nimrod` env active:

```bash
bash .devcontainer/install_dependencies_macos.sh
```

This installs `espeak` and `boost` via Homebrew, installs the `kenlm` Python
bindings, and prints the `PHONEMIZER_ESPEAK_LIBRARY` line to add to your shell
profile. It needs [Homebrew](https://brew.sh) and does **not** use `sudo`.

## Troubleshooting

- **`ModuleNotFoundError` for lightning/hydra/torchaudio/etc.** ‚Äî you're almost
  certainly in the wrong env. Check with `echo $CONDA_DEFAULT_ENV` (should be
  `nimrod`, not `base`) and `which python`.
- **"A module compiled using NumPy 1.x cannot be run in NumPy 2.x"** ‚Äî numpy got
  upgraded past the pin. Fix: `pip install "numpy<2"`.
- **torchtext fails to install / no wheel** ‚Äî confirm torch is exactly `2.3.0`
  and Python is `3.9`‚Äì`3.12`. torchtext has no wheels for newer torch or
  Python 3.13+.
- **`phonemizer` can't find espeak** ‚Äî set
  `export PHONEMIZER_ESPEAK_LIBRARY=...` (printed by the install script) in your
  shell profile.
- **Stale/partial `nimrod` env** (e.g. an older one with torch 2.12) ‚Äî recreate
  it cleanly: `micromamba env remove -n nimrod` then redo the TL;DR.
