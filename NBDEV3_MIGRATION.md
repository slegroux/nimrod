# nbdev 2 → 3 migration plan (deferred)

Status: **not started** — deliberately deferred. The repo currently runs on
**nbdev 2** (`settings.ini`). This is a separate, deliberate task; nothing here
is required to test or run the project. Do it when we want to modernize the
build/config, not under time pressure.

## Why deferred

nbdev 3 is a config/plumbing change, not a feature we need right now. It touches
the build glue (`setup.py`), pre-commit, and CI. Migrating mid-stream adds churn
and risk to a working nbdev-2 project for zero testing benefit. Migrate on its
own branch later.

**Current state this plan moves away from:** nbdev is pinned to a known-good 2.x
set in `settings.ini` `dev_requirements` —
`nbdev==2.3.13 fastcore==1.5.29 execnb==0.1.5 setuptools<81` (latest `execnb`
dropped `execnb.nbio` and latest `fastcore` dropped `fastcore.dispatch`, both of
which nbdev 2.x imports; `setuptools<81` keeps `pkg_resources`). CI installs this
via `.[dev]` and skips `image.med` + `audio.datasets.stt`. The migration replaces
that pin with nbdev 3.

## What nbdev 3 actually changes

Sourced from the nbdev docs + changelog (see Sources). Confirmed against the
`nbdev 3.0.17` we briefly installed.

1. **Config moves `settings.ini` → `pyproject.toml`** (PEP 621). Project metadata
   goes in `[project]`; nbdev settings go in `[tool.nbdev]`. nbdev 3 hard-errors
   if `settings.ini` is still present.
2. **CLI commands: underscores → hyphens** (`nbdev_test` → `nbdev-test`,
   `nbdev_clean` → `nbdev-clean`, `nbdev_export` → `nbdev-export`). The 3.0.17
   install shipped **only** hyphenated entry points — no underscore aliases — so
   every underscore reference breaks.
3. **`setup.py` becomes obsolete.** With `[project]` metadata in `pyproject.toml`,
   the custom nbdev-2 `setup.py` (which parses `settings.ini`) is deleted; the
   build backend reads `pyproject.toml` directly.
4. **Notebook content is unchanged.** `#|export` / `#|hide` directives and the
   `nimrod/` modules need no edits. This is the low-risk part.
5. `nbdev-migrate-config` automates the `settings.ini → pyproject.toml` conversion
   and (re)creates the lib `__init__.py`. Per the migrate API docs it does **not**
   modify `setup.py`, pre-commit, or CI — treat those as manual. (The
   getting-started page claims it also updates GitHub Actions; this is
   contradicted by the API docs, so **verify** rather than assume.)

## Repo-specific blast radius

Files in *this* repo that reference nbdev-2 config or underscore CLI:

| File | What's there now | Migration action |
|------|------------------|------------------|
| `settings.ini` | all config + `requirements`/`dev_requirements` | converted to `pyproject.toml`, then **deleted** |
| `setup.py` | reads `settings.ini` via ConfigParser | **deleted** (obsolete). Makes our recent `pkg_resources`→`packaging` edit moot |
| `pyproject.toml` | only `[build-system]` (+ our `packaging` req) | gains `[project]` + `[tool.nbdev]`; `packaging` build-req can be dropped |
| `.pre-commit-config.yaml` | `entry: nbdev_clean` | → `nbdev-clean` |
| `settings.ini` `dev_requirements` | `nbdev==2.3.13 fastcore==1.5.29 execnb==0.1.5 setuptools<81` | drop the nbdev/fastcore/execnb pins (v3 manages compatible versions) and `setuptools<81`; carry over to `[project.optional-dependencies].dev` |
| `.github/workflows/test.yaml` | `nbdev_clean`, `nbdev_export`, `nbdev_test … --skip_file_re '…image\.med\|audio\.datasets\.stt'`; installs `.[dev]` | rename all three to hyphens; keep the skip list; confirm `[dev]` extra still resolves from `[project.optional-dependencies]` |
| `.github/workflows/deploy.yaml` | `fastai/workflows/quarto-ghp@master` | reusable fastai workflow — should track nbdev 3; **verify** docs build |
| `nbs/_quarto.yml`, `nbs/sidebar.yml` | quarto/sidebar config | may be regenerated by `nbdev-docs`/`nbdev-sidebar`; diff after migration |
| `INSTALL.md` | points deps at `settings.ini` | update prose to `pyproject.toml` |
| `Makefile` | no nbdev CLI calls (only `pytest`) | no change |

`requirements` and `dev_requirements` from `settings.ini` map to
`[project.dependencies]` and `[project.optional-dependencies].dev`. The current
torch matrix (`torch==2.9.1 …`, `transformers<5`) carries over verbatim.

## Procedure (when we do it)

1. **Branch + backup.** `git checkout -b nbdev3-migration`; keep `settings.ini`
   and `setup.py` in history (they'll be deleted, recoverable via git).
2. **Upgrade tooling.** `pip install "nbdev>=3"` (also bumps `fastcore`/`execnb`).
3. **Run the converter.** `nbdev-migrate-config` in repo root → writes `[project]`
   + `[tool.nbdev]` into `pyproject.toml`, creates `__init__.py`.
4. **Review `pyproject.toml`.** Confirm metadata (version, lib_name=`slg_nimrod`,
   lib_path=`nimrod`, nbs_path=`nbs`, `tst_flags=notest`, deps, `[dev]` extra),
   and that `[build-system]` still builds. Drop the now-unneeded `packaging`
   build-req.
5. **Delete obsolete files.** `git rm settings.ini setup.py`.
6. **Fix underscore CLI refs** (the converter likely misses these):
   - `.pre-commit-config.yaml`: `nbdev_clean` → `nbdev-clean`
   - `.github/workflows/test.yaml`: `nbdev_clean`/`nbdev_export`/`nbdev_test` → hyphens
7. **Update `INSTALL.md`** deps prose: `settings.ini` → `pyproject.toml`.
8. **Verify** (below). Iterate until green.
9. **PR** separately from any dependency-pin work.

## Verification checklist

- [ ] `pip install -e ".[dev]"` succeeds from `pyproject.toml` (no `setup.py`).
- [ ] `python -c "import nimrod"` works; version in `nimrod/__init__.py` matches.
- [ ] `nbdev-export` produces **no** git diff under `nimrod/` (lib in sync).
- [ ] `nbdev-clean` runs (pre-commit hook passes).
- [ ] `nbdev-test` runs the suite (same pass/fail set as nbdev 2 baseline).
- [ ] `nbdev-docs` / quarto build succeeds locally; `deploy.yaml` green on CI.
- [ ] `test.yaml` green on CI.

## Rollback

Everything is one branch + git-tracked deletions. To abort:
`git checkout main && git branch -D nbdev3-migration`, then `pip install "nbdev<3"`.
